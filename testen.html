Testen in der Universität

In der Universität wurde einem das Testen mehr oder weniger nur theoretisch erklärt, denn praktisch gab es nur wenige Möglichkeiten dies anzuwenden. Selbst in Kursen, wo vergleichsweise viel programmiert werden muss, wird das schreiben von Tests vernachlässigt.

Warum? Relativ einfach: Tests bringen keine gute Noten. Für einen Studenten ist der Nutzen von Tests effektiv 0, da sie keine Ergebnisse produzieren, die vom Dozenten oder vom Lehrstuhl bewertet werden können.

Unit-Tests

Unit-Tests sind die erste Stufe mit der man in Berührung kommt. Es muss auch geklärt werden, was als "Unit" innerhalb einer Programmiersprache gilt und für gewöhnlich versteht man folgendes: Die kleinsten prüfbaren Einheiten, welche man Ausführen kann.

Wie viele Unit-Tests ich in der Universität geschrieben habe? Ähm... Null? Wenn man bedenkt, wie man ganz schnell ganz viele solcher Tests benötigt, kann an einem einfach Beispiel gezeigt werden. 

public static boolean isPrime(long number) {
	if (number <= 1) {
		return false;
	} else if (number == 2) {
		return true;
	} else if ((number & 1) == 0) {
		return false;
	} else {
		for (long div = 3; div * div <= number; div += 2) {
			if (number % div == 0) {
				return false;
			}
		}
		return true;
	}
}

Auf dem ersten Blick erscheint jedem sofort erklärlich, was diese Methode tut: Sie überprüft, ob die übergebene Zahl eine Primzahl ist. Doch dafür müssen folgende allgemeine Prüffälle her:

  • Eine Zahl, die kleiner oder gleich 1 ist.
  • Die Zahl 2, da sie die einzig gerade Primzahl ist.
  • Eine gerade Zahl größer als zwei.
  • Eine ungerade Zahl, die durch eine andere kleinere ungerade Zahl teilbar ist.
  • Eine ungerade Zahl, die NICHT durch eine andere kleinere ungerade Zahl teilbar ist.
Eine mögliche Menge an Zahlen, die diese Punkte erfüllt, wäre zum Beispiel 1, 2, 3, 4 und 9. Und diese Menge deckt gerade alle Anweisungen dieser Methode ab! Anders ausgedrückt, sie erreicht das C0-Kriterium. Für das C1-Kriterium fehlt jedoch eine Zahl, bei dem die if-Abfrage innerhalb der Schleife mindestens einmal zu false ausgewertet wird. Die erste Zahl dafür wäre 11 und die Menge enthält jetzt sechs Zahlen. Das C2-Kriterium lässt sich nur in der Variante b oder c erreichen, da die Variante a bei Schleifen unmöglich wird. Mit den vorherigen Zahlen ist bereits ein Teil der b-Variante erfüllt: Alle Pfade, die bei einem einmaligen Schleifendurchlauf auftreten können, sind abgedeckt. Um alle Pfade bei einem zweimaligen Durchlaufen abzudecken, brauchen wir zwei weitere Zahlen: Als erstes 25, da diese durch 5 aber nicht durch 3 teilbar ist. Und 29, die erste Primzahl, die größer als 25 ist. Es kommen somit zwei weitere Zahlen hinzu und die Menge enthält jetzt acht Zahlen.
